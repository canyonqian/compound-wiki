## 背景

用户遇到的三大痛点：
1. 每次都要重新交代背景 — 写周报、PPT 都要从零介绍自己
2. AI 记不住你的偏好 — 喜欢的风格说一次忘一次
3. 用得越多越觉得它不懂你 — 教过 N 次下次还是老样子

**核心结论**: "这不怪 AI，是没给它'记忆'"

## 三层记忆系统

### Layer 1: 知识图谱 (/life/areas/)
- 记录实体信息（谁/什么东西）+ 动态摘要
- 类比: 给新同事发入职手册
- 数据结构: items.json（原子事实）+ summary.md（动态摘要）

### Layer 2: 每日笔记 (memory/YYYY-MM-DD.md)
- 每天发生了什么，事件日志
- 类比: 老秘书知道你的习惯

### Layer 3: 隐性知识 (MEMORY.md)
- 模式、偏好、经验教训
- 类比: 自己越做越熟练

## 核心技术: 原子事实的替代机制

这是最牛的设计——**替代而非删除**：

```json
// 旧事实被标记为 superseded，不直接删除
{
  "id": "formula-001",
  "status": "superseded",
  "supersededBy": "formula-002",
  "supersededAt": "2026-02-01"
}
```

优势: AI 可以追溯偏好如何随时间演变——哪些曾经有效、为什么被替代。

## 复利引擎循环

```
对话(写作) → 事实提取(Hook自动) → 知识图谱增长 
→ 每周综合整理 → 更好回复 → 回到对话...
```

## 时间线效果

| 时间 | AI 能力 |
|------|--------|
| 第 1 周 | 记住基本背景 |
| 第 1 月 | 记住任务偏好 |
| 第 3 月 | 记住哪些风格效果好 |
| 半年+ | 效率翻倍，AI 比你还懂你 |
