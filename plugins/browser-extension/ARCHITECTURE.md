## 核心能力

### 🧠 智能内容检测（不需要你判断）

系统自动判断一篇文章是否值得保存，基于 **6 个维度**：

| 维度 | 权重 | 说明 |
|------|------|------|
| **内容密度** | 25% | 正文字数 vs 总字数（过滤导航/广告页） |
| **阅读深度** | 20% | 滚动比例（扫一眼 vs 读完） |
| **停留时长** | 20% | 在页面的有效阅读时间 |
| **来源可信度** | 15% | 域名白名单/黑名单/已知质量站点 |
| **交互信号** | 10% | 高亮文字、复制操作、返回重读 |
| **主题相关性** | 10% | 与已有Wiki内容的语义关联 |

### 📊 价值评分阈值

```
score >= 80: ⚡ 立即自动捕获（高质量长文+读完）
score >= 60: 🔍 观察模式（30秒后重评，可能捕获）
score < 40:  ❌ 忽略（首页/导航/短内容）
```

### 🔄 全自动化流程

```
你打开一篇技术文章
        ↓
  Content Script 开始分析（后台静默）
        ↓
  你正常阅读文章...
  [系统在后台记录：滚动位置、停留时间、是否有高亮]
        ↓
  你读完了，切换到下一个标签页
  [系统检测到页面失焦或关闭]
        ↓
  最终评分：内容密度92 + 阅读深度88 + 停留85 = 86分
  超过阈值 → 自动捕获！
        ↓
  提取：标题、作者、发布日期、正文、URL
  清洗：去掉导航栏、广告、评论、footer
  分类：技术文章 → tag: [tech, ai, llm]
        ↓
  POST 到本地 Wiki 服务器
        ↓
  Pipeline 自动接收 → 存入 raw/ → LLM INGEST → Wiki页面生成
        ↓
  浏览器右下角弹出："🧠 已存入 Wiki：xxx文章"
        ↓
  完事！你没做任何操作。
```

## 支持的浏览器

- **Chrome / Edge / Brave** (Manifest V3) — ✅ 完整支持
- **Firefox** (Manifest V2/V3 兼容) — ✅ 支持
- **Arc** — ✅ 基于 Chromium，支持

## 与旧版 Bookmarklet 的区别

| | 旧版 Bookmarklet | 新版智能扩展 |
|--|-----------------|-------------|
| 触发方式 | 手动点击按钮 | **全自动** |
| 内容判断 | 无（全靠人脑） | **6维智能评分** |
| 阅读感知 | 无 | **追踪阅读行为** |
| 正文提取 | 原始innerText | **Readability算法** |
| 去噪 | 无 | **去广告/导航/footer** |
| 分类 | 无 | **自动分类+标签** |
| 通知 | alert弹窗 | **静默气泡通知** |
